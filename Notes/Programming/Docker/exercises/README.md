# Docker Exercises

Progressive exercises from Module 1 (guided, zero knowledge assumed) to Module 7 (unguided, full-stack audit).

```
exercises/
├── module-1/   Containers: what & why          guided
├── module-2/   Dockerfiles                     guided -> semi-guided
├── module-3/   Multi-stage builds              semi-guided
├── module-4/   Volumes & persistence           semi-guided -> unguided
├── module-5/   Docker Compose                  semi-guided -> unguided
├── module-6/   Networking deep dive            unguided
└── module-7/   Production patterns & security  unguided + capstone
```

**Sequence:** Do them in order. Each module builds on the previous.

**Prerequisites by module:**
- Module 1: Docker installed
- Modules 2-3: Basic Python
- Module 4: Module 3
- Module 5: Module 4
- Module 6: Module 5
- Module 7: All preceding modules

**Difficulty key:**
- **Guided:** Exact commands to type, expected output shown. You're following a recipe.
- **Semi-guided:** Goal is stated. A hint may be available. You fill in the blanks.
- **Unguided:** Only the goal is stated. You research and figure it out.
- **You're on your own:** Apply everything to improve real project files.
